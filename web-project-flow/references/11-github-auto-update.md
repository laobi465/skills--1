# 网站项目 GitHub 自动更新管理系统

你是一名资深全栈开发工程师，精通 GitHub CI/CD、Webhook 自动化、后端服务运维与前端管理后台开发。
请基于以下需求，为我设计并实现一套完整的「网站项目 GitHub 自动更新管理系统」。

## 核心功能需求

### 1. GitHub 自动更新机制

- 配置 GitHub Webhook 监听仓库 push 事件（指定分支，如 main/master）
- 收到 Webhook 触发后，自动执行 `git pull` 拉取最新代码
- 代码拉取完成后自动执行依赖安装（如 `composer install` / `npm install` / `pip install` 等）
- 自动执行数据库迁移（如存在 migration 文件）
- 自动清理缓存（OPcache、路由缓存、配置缓存等）

### 2. 更新后自动重启

- 代码更新完成后，自动重启网站服务进程
- 支持多种运行环境适配：
  - Docker 容器环境：重启对应容器或执行 `docker-compose restart`
  - 宝塔面板 / PHP 环境：重载 PHP-FPM、重启 Nginx/Apache
  - Node.js / Python 项目：重启 PM2 / systemd 管理的服务进程
- 重启前进行健康检查，重启后验证服务可用性
- 重启失败自动回滚到上一版本，确保服务不中断

### 3. 管理员弹窗提醒

- 网站后台实时检测是否有新版本可用
- 当检测到新版本时，已登录的管理员账号在页面上弹出醒目提醒弹窗
- 弹窗内容包含：新版本号、更新内容摘要、更新时间
- 弹窗提供「立即更新」和「稍后提醒」两个操作按钮
- 使用 WebSocket / 轮询机制实现实时推送，确保所有在线管理员同步收到提醒

### 4. 管理员后台更新面板

- 在后台首页或系统设置页面的显眼位置放置「系统更新」模块
- 展示信息：
  - 当前版本号（当前部署的 commit hash / tag）
  - 最新可用版本号
  - 更新内容列表（changelog，从 GitHub release 或 commit message 提取）
  - 更新时间与发布者
- 提供「立即更新」按钮，点击后手动触发更新流程
- 更新过程中显示进度条与实时日志输出
- 更新完成后显示成功/失败状态与结果详情
- 提供版本历史记录，支持查看过往更新日志

## 技术要求

### 安全机制

- Webhook 端点使用密钥签名验证（GitHub Secret），防止非法触发
- 更新接口仅限超级管理员权限访问
- 所有更新操作记录审计日志（操作人、时间、IP、更新前后版本）
- 更新脚本执行权限最小化，禁止直接暴露 shell 执行接口

### 可靠性保障

- 更新前自动备份代码与数据库
- 支持一键回滚到上一个稳定版本
- 更新过程加锁，防止并发重复触发
- 异常情况自动告警（邮件 / 站内通知）

### 适配性

- 代码结构清晰，易于集成到现有项目中
- 配置项独立，支持通过环境变量或配置文件调整
- 提供详细的部署文档与使用说明
